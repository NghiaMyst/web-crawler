## Spacing Scale

Tailwind v4 uses the default spacing scale (multiples of 4px). No custom `tailwind.config.js` exists — all customization lives in `globals.css @theme`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`p-1`, `gap-1`) | Icon gaps, inline padding |
| sm | 8px (`p-2`, `gap-2`) | Compact element spacing, badge padding |
| md | 16px (`p-4`, `gap-4`) | Default element spacing, cell horizontal padding |
| lg | 24px (`p-6`, `gap-6`) | Card padding, section padding |
| xl | 32px (`p-8`, `gap-8`) | Layout gaps, hero padding |
| 2xl | 48px (`p-12`) | Empty state vertical padding |
| 3xl | 64px | Page-level vertical spacing |

**Exceptions:**
- Sidebar width: 220px fixed (maintained from current Sidebar.tsx implementation)
- Hero search input: `padding: 16px 24px` (multiples of 4 — both axes)
- Category tile grid: `gap-16` (16px), `gridTemplateColumns: repeat(4,1fr)` per Variant B wireframe

**Note on table row padding:** Table rows use `py-3` (12px top + 12px bottom = 24px cell padding), producing an effective row height of approximately 44px — satisfies D-04 comfortable density. The previous `py-3.5` (14px, non-multiple of 4) has been removed.

**Source:** Variant B wireframe (wireframes.jsx lines 169–217), CONTEXT.md D-04, globals.css base layer

## Typography

**Font assignment rule:**
- Plus Jakarta Sans (`font-heading`) — used for hero headline, display numerics, page titles, card headings, the "webcrawler" logotype in sidebar
- Inter (`font-sans`) — used for all body text, table cells, labels, badges, inputs, button text, meta/secondary copy

**Base font size:** 17px (declared on `html { font-size: 17px }` in globals.css `@layer base`)

**Exactly 4 type roles. Exactly 2 declared weights.**

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Display | Plus Jakarta Sans | 28–32px (`text-3xl` / `text-4xl` at 17px base) | 700 (Heavy) | 1.1 | Hero heading "what did we crawl today?", page-level display numbers, logo/brand "webcrawler" |
| Heading | Plus Jakarta Sans | 20–22px (`text-xl` / `text-2xl`) | 700 (Heavy) | 1.2 | PageHeader title, card section headings, "just crawled" strip heading |
| Body | Inter | 16px (`text-base`) | 400 (Regular) | 1.5 | Table cell content, description text, modal body, form inputs |
| Label | Inter | 13px (`text-xs` at 17px base ≈ 12.75px, use `text-xs`) | 400 (Regular) | 1.4 | Badges, filter chips, meta copy, timestamps, source URLs, captions, empty state sub-copy |

**Declared weights:**
- Heavy: 700 — Display, Heading
- Regular: 400 — Body, Label

**Utility-class exception (not a type role):** Tailwind `font-semibold` (weight 600) is used for sub-level emphasis within Body context — nav active state, table header labels, category tile labels, modal section labels, chart card titles. This is a utility-class override, not a type scale entry. It does not count as a declared weight role.

**Specific copy sizes from Variant B wireframe (component-level overrides, not scale entries):**
- Logo / brand "webcrawler": 38px, weight 700, Plus Jakarta Sans (`text-4xl font-bold font-heading`)
- Hero headline: 46px, weight 700, line-height 1.0, Plus Jakarta Sans
- Hero search input placeholder: 28px, Plus Jakarta Sans (current HeroSearchInput.tsx already implements this)
- Category tile label (`h3`): 16px, weight 700 (`font-bold`), Plus Jakarta Sans — override on Body size, listed here as component contract, not a new scale role
- Category tile count: 13px, Inter — aligns with Label role

**Source:** wireframes.jsx VariantB lines 154–249, globals.css `@layer base`, layout.tsx

## Playwright Snapshot Strategy

**Playwright is not yet installed.** Wave 0 task required before any screenshot tests.

| Property | Value |

**playwright.config.ts pattern:**
```typescript

    url: 'http://localhost:3000',
```

## Anti-Patterns

Do not do any of the following during implementation:

1. **Do not add new CVA variants to badge.tsx** — use `variant="outline"` + className override (Shadcn v4 uses @base-ui/react internals)
2. **Do not modify VolumeChart CHART_COLORS array** — it already reads `var(--chart-*)` correctly; only wrap chart sections in card divs
3. **Do not forget MobileNav.tsx** — has 2 dark background references and 3 coral text references; all must be updated simultaneously with Sidebar.tsx
4. **Do not put raw OKLCH values in `@theme inline` block** — the `@theme inline` block only contains `var(--token)` bridge references; raw values go in `:root {}`
5. **Do not run `npx playwright test` before `--update-snapshots`** — first run must generate baselines or all tests fail
6. **Do not apply hero section to non-entry pages (D-03)** — hero lives only on `/entries`; all other pages use `PageHeader`
7. **Do not introduce dark mode styles** — dark mode is deferred out of scope
8. **Do not use `py-3.5` (14px)** — table row padding must use `py-3` (12px) or `py-4` (16px); all spacing values must be multiples of 4px
9. **Do not use "Save" or "Cancel" as bare modal button labels** — use "Save Source" / "Save Rule" for primary actions; use "Discard Changes" or "Close" for secondary actions per context
