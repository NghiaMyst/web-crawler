<objective>
Extend `GET /api/entries` with an optional `?q=` query parameter that filters entries by PostgreSQL full-text search against `data_entries.search_vector`. When `q` is absent/empty, the endpoint behaves identically to today (no FTS filter applied). When `q` is non-empty, EF Core generates a `WHERE search_vector @@ plainto_tsquery('english', @q)` clause.

Purpose: Closes SC-3 (search API ?q= param). Wires the FTS infrastructure from Plan 02 into the dashboard's API contract. Sort by date is preserved (D-07: no relevance ranking). Cursor pagination is preserved (FTS chained before `.Take(limit+1)`). Uses `PlainToTsQuery` (not `ToTsQuery`) per security requirement — raw user input cannot inject `and-or-not` FTS operators.

Output: Modified `EntriesEndpoints.GetEntries` with `q` parameter and FTS filter, plus Wave 0 EntriesSearchTests.cs stubs marked as Integration (skipped in unit test runs because they require a real PostgreSQL).
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-03-01 | Tampering | FTS operator injection via `?q=` | mitigate | Use `EF.Functions.PlainToTsQuery("english", q)` exclusively. PlainToTsQuery ignores `and-or-not-paren` punctuation, treating user input as a literal phrase. ToTsQuery (which interprets those as FTS operators) is forbidden — enforced by grep in acceptance criteria. |
| T-11-03-02 | Tampering | SQL injection via `?q=` | mitigate | EF Core parameterizes `q` through Npgsql — the value goes into the SQL as `$N` placeholder, never string-interpolated. Verifiable via EF Core query logging. |
| T-11-03-03 | Denial of Service | Long `q` values causing expensive FTS | accept | PlainToTsQuery is bounded by PostgreSQL tsquery limits. The existing endpoint has `limit = Math.Clamp(limit, 1, 100)` and the GIN index makes the FTS lookup logarithmic. No additional rate limiting in Phase 11. |
| T-11-03-04 | Information Disclosure | FTS results leaking unindexed fields | accept | search_vector contains only fields admin-listed in search_configs.json_paths. Response payload is unchanged — entries still return their full JSONB payload as today. No new disclosure surface. |
| T-11-03-05 | Tampering | Cursor manipulation combined with q | accept | Existing cursor decoding has try/catch around invalid base64 → returns first page. Adding q does not change cursor security: q is applied before cursor decoding, so a malformed cursor still falls back safely. |
</threat_model>
