## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, gap between connection dot and wordmark |
| sm | 8px | Gap between filter controls, button padding compact |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, sidebar nav group padding |
| xl | 32px | Layout gaps between page sections |
| 2xl | 48px | Empty-state vertical padding (py-16 = 64px — see exception) |
| 3xl | 64px | Page-level top/bottom breathing room |

Exceptions:
- Empty-state blocks use `py-16` (64px) — matches existing `JobsEmptyState` and `AlertsEmptyState` patterns. Do not reduce to 48px.
- Nav search input height: 32px (`h-8`) — matches existing `Input` component default and `entries-filters.tsx` input sizing.
- Sidebar brand row height: 56px (`h-14`) — matches `Sidebar.tsx` and `MobileNav.tsx` existing header height. Search input must fit within this row or be placed directly below it in a 40px secondary band.

Source: existing component patterns (`JobsEmptyState`, `AlertsEmptyState`, `MobileNav.tsx`, `Sidebar.tsx`, `entries-filters.tsx`).

## SearchInput Interaction Contract

Location: `components/search/SearchInput.tsx` — `'use client'`

**Trigger:** Enter key on the input field.

**Behavior on Enter:**
1. Read current `searchParams` (via `useSearchParams`).
2. Build new `URLSearchParams` from current params.
3. Delete `cursor` key (reset pagination).
4. If `value.trim()` is non-empty: `params.set('q', value.trim())`.
5. If `value.trim()` is empty: `params.delete('q')`.
6. Call `router.push(\`/entries?${params.toString()}\`)` inside `startTransition`.

**Preserve on navigation:** `category` and `sourceId` params are carried forward (they stay in URLSearchParams unless the user clears them separately via `EntriesFilters`).

**Input states:**
- Default: placeholder "Search entries…" — `text-muted-foreground` color, inherited from `Input` component.
- Active / focused: border becomes `--ring` color, ring shown — inherited from `Input` component styles.
- Pre-filled: when the current URL contains `?q=`, the input mounts with `defaultValue` equal to `searchParams.get('q')` (use controlled `value` + `useState` initialized from `searchParams.get('q') ?? ''`).
- Loading / transitioning: no spinner on the input itself. The `isPending` state from `useTransition` may be used to set `aria-busy="true"` on the input.

**Keyboard:** Enter submits. Escape clears the input value (does NOT navigate — only clears local state). Tab moves to next focusable element.

**Placement in Sidebar (desktop):**
```
<aside> (md:flex md:flex-col md:w-[240px] bg-zinc-900)
  <div> brand row (h-14, border-b) — "Web Crawler" + ConnectionDot
  <div> search band (px-3 py-2) — <SearchInput />
  <NavLinks />
</aside>
```
Search band: `px-3 py-2` (12px horizontal, 8px vertical). Input fills full width of the band.

**Placement in MobileNav (SheetContent):**
```
<SheetContent side="left" w-[260px] p-0>
  <SheetHeader> (h-14 border-b) — "Web Crawler"
  <div> search band (px-3 py-2) — <SearchInput />
  <NavLinks onNavigate={...} />
</SheetContent>
```

## Keyword Highlight Contract

File: `components/entries/entries-table.tsx`

**When `q` prop is absent or empty:** `formatPayloadPreview` returns a plain `string` (current behavior — no change).

**When `q` prop is present and non-empty:** `formatPayloadPreview` accepts `q` and returns `React.ReactNode`. Matched tokens (case-insensitive, literal string match — no stemming) are wrapped in `<mark className="bg-yellow-200 dark:bg-yellow-800/50 rounded-sm px-0.5">{part}</mark>`.

**Regex escaping:** Escape `q` before inserting into RegExp: `q.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')`. Use `new RegExp(\`(${escaped})\`, 'gi')` for case-insensitive split.

**Scope:** Only the payload preview text (first 3 keys, truncated to 30 chars each). Do NOT apply highlight to Entry Key or Category columns.

**No server-side `ts_headline`:** Client-side mark is the final approach (see RESEARCH.md Pattern 4 for rationale — XSS safety, zero API shape change).

## Active Search Filter Badge Contract

File: `components/entries/entries-filters.tsx`

When `searchParams.has('q')` is true, render after the existing filter controls:

```
<Badge variant="secondary" className="flex items-center gap-1 text-xs">
  <Search size={12} />  {/* lucide-react */}
  Searching: {q}
  <button aria-label="Clear search" onClick={clearQ}>
    <X size={12} />
  </button>
</Badge>
```

`clearQ` calls `handleChange('q', null)` (follows existing `createQueryString` pattern which deletes null values).

The `hasFilters` check in `EntriesFilters` must include `searchParams.has('q')` so the Reset button appears when a search is active.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (search nav input label) | `aria-label="Search entries"` (no visible label — input is self-describing via placeholder) |
| Search input placeholder | `Search entries…` |
| Empty state heading (no results for query) | `No entries found` |
| Empty state body (no results for query) | `No results for "{q}". Try a different search term or clear filters.` |
| Empty state body (no q, no entries) | `No entries found. Adjust filters or wait for new crawl data.` (existing — unchanged) |
| Active search badge | `Searching: {q}` |
| Clear search button label | `aria-label="Clear search"` |
| Error state (API failure on search) | `Search failed. Check your connection and try again.` |

Notes:
- The empty state with `q` present must display the query in quotes so the user sees what was searched.
- There is no destructive action in the Phase 11 search UI. The "clear search" action is non-destructive (navigates to `/entries` without `q`).

Source: upstream artifacts (CONTEXT.md D-09, D-10) + existing `EntriesTable` empty state copy pattern.

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Search input | `type="search"` attribute (enables browser native clear X on webkit), `aria-label="Search entries"` |
| Search band in Sidebar | Must be reachable via Tab from the brand row; `SearchInput` is the next focusable element |
| Clear search button | `aria-label="Clear search"`, `type="button"` (not submit), minimum 32x32px touch target |
| Highlight `<mark>` elements | No additional `aria-label` needed — mark is a semantic HTML element, screen readers announce highlighted content |
| Active search badge | Not interactive beyond the X button; badge text is readable by screen readers |
| `isPending` state | Set `aria-busy="true"` on the `<Input>` when `isPending` is true |
