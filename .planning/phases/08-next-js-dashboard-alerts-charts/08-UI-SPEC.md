## Spacing Scale

Declared values (multiples of 4 only). Source: Tailwind v4 defaults + Phase 7 observed usage in SourcesTable.tsx, SourceModal.tsx, NavLinks.tsx.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge inner padding |
| sm | 8px | Compact element spacing, button icon padding |
| md | 16px | Default element spacing, form field gaps (`space-y-4`) |
| lg | 24px | Section padding, modal content gaps |
| xl | 32px | Layout gaps, page content padding |
| 2xl | 48px | Empty state vertical padding (`py-16` = 64px — see exception below) |
| 3xl | 64px | Page-level spacing |

**Exceptions:**

- Nav link touch target: min-height 44px (`min-h-[44px]`) — verified in NavLinks.tsx. Apply identically to all nav links added for `/alerts`, `/notifications`, `/charts`.
- Empty state vertical padding: `py-16` (64px top + 64px bottom) — verified in SourcesEmptyState.tsx. Use for all three new empty states.
- Chart container height: explicit `h-64` (256px) minimum wrapping div required by Recharts `ResponsiveContainer`. See Interaction Contracts section.

## Color

All values from globals.css CSS custom properties. The palette is neutral (zero-chroma grays) with one destructive semantic color.

| Role | Light Mode Value | Dark Mode Value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) — background | `oklch(1 0 0)` (white) | `oklch(0.145 0 0)` (near-black) | Page background, modal background |
| Secondary (30%) — surfaces | `oklch(0.985 0 0)` (zinc-50) | `oklch(0.205 0 0)` (zinc-900) | Sidebar, table header (`bg-zinc-100`), card backgrounds |
| Accent (10%) — primary actions | `oklch(0.205 0 0)` (zinc-900) | `oklch(0.922 0 0)` (zinc-200) | Primary buttons, active nav item background |
| Destructive | `oklch(0.577 0.245 27.325)` (red-600) | `oklch(0.704 0.191 22.216)` (red-500) | Destructive action buttons only |

**Accent reserved for (explicit list — not all interactive elements):**

1. Primary submit button in modals ("Save Alert Rule", "Save")
2. Active nav item background (`bg-zinc-900 text-white`)
3. Focused ring on inputs and buttons (`--ring: oklch(0.708 0 0)`)

**Accent must NOT be used for:** hover states (use `hover:bg-zinc-200`), secondary actions, icon buttons, filter dropdowns, or chart lines.

## Visual Hierarchy

### `/alerts` — Alert Rules Page

The primary focal anchor is the "Add Alert Rule" button in the top-right of the page header. The eye moves: page heading (top-left, `text-xl font-semibold`) → "Add Alert Rule" CTA (top-right, accent button) → table body (center). The table is the content anchor; condition type badges in column 3 are the secondary visual interest point. Row action icons (Pencil, Trash2) recede until hover.

### `/notifications` — Notification History Page

The interaction anchor is the source filter dropdown, positioned above the table and right-aligned (matching the entries page pattern). The eye moves: page heading → filter dropdown → table. Status badges ("Sent" / "Failed") in column 1 carry the strongest color contrast within each row and draw scanning attention — green-600 for sent, red-500 for failed. The message column (widest) carries the most text weight but is secondary to status.

### `/charts` — Charts Page

The primary visual anchor is the line chart (Entries over time), which occupies the full content width at `h-64` minimum and is the first element in the content area. The date range selector (top-right of the chart section, small `<Select>`) is the secondary interaction anchor. The stacked bar chart below the line chart is a supporting element and draws the eye second. The chart legend (auto-generated by `ChartLegend`) sits below each chart and provides source identity context — not a primary anchor.

## Interaction Contracts

### Alert Rules Page — CRUD Interactions

**Pattern:** Mirror SourcesClient/SourceModal/DeleteSourceDialog exactly. No new patterns introduced.

1. **Add flow:** "Add Alert Rule" button → AlertRuleModal (add mode) opens → form submitted → `createAlertRuleAction` → `useOptimistic` prepends row with `_pending: true` → server confirms → row updates to real data.
2. **Edit flow:** Row "Edit" button → AlertRuleModal (edit mode) opens with pre-populated data → form submitted → `updateAlertRuleAction` → `useOptimistic` marks row `_pending: true` → server confirms → row updates.
3. **Delete flow:** Row "Delete" button → DeleteAlertDialog opens → "Delete Alert Rule" confirm → `deleteAlertRuleAction` → `useOptimistic` removes row optimistically → server confirms.
4. **Rollback:** On `{ ok: false }` from any action, inline `<p role="alert">` inside modal displays `result.error`. Table row reverts to pre-optimistic state.
5. **Pending indicator:** Row with `_pending: true` renders `opacity-60` + `<Loader2 size={14} className="animate-spin" aria-label="Saving" />` next to name — identical to SourcesTable pattern.

### Alert Condition Form — Conditional Field Reveal

1. Condition type selector defaults to `new_item` on add mode.
2. Switching to `field_changed`: "Field path" input fades in (`animate-in fade-in-0` or simple conditional render — no animation required).
3. Switching to `threshold`: "Field path" AND "Threshold value" inputs appear.
4. Switching back to `new_item`: extra fields unmount (form state reset for those fields via `resetField`).
5. Validation is live (on blur) for text fields, on submit for the full form.

### Notification History Page — Filter Interaction

1. Source filter dropdown renders above the table, right-aligned (matches filter position in Entries page).
2. Selecting a source updates URL search param `?sourceId=uuid` via `router.push` — no full page reload.
3. "All sources" option clears the `sourceId` param.
4. Filter state persists on page refresh (URL-driven, not `useState`).

### Charts Page — Date Range Interaction

1. Date range dropdown defaults to `7d` on mount.
2. Selecting a new range triggers a client-side fetch to `GET /api/stats/volume?groupBy=day&range={range}`.
3. During fetch: chart area shows `<Skeleton className="h-64 w-full" />`.
4. Data is pivoted from long-format API rows to wide-format Recharts objects in a `pivotVolumeData()` function (see RESEARCH.md Pattern 5).
5. Each source gets one `<Line>` (line chart) or `<Bar>` (bar chart) using `dataKey={sourceName}` and color from `ChartConfig`.
6. Single `/charts` page layout: line chart (volume over time) rendered first, followed by stacked bar chart (per-source breakdown) in a separate section. No separate routes.

### Loading States

All three new pages use `loading.tsx` Next.js convention OR inline Skeleton components:

| Page | Loading pattern |
|------|----------------|
| `/alerts` | Skeleton table: 5 rows of `<Skeleton className="h-10 w-full" />` |
| `/notifications` | Skeleton table: 5 rows of `<Skeleton className="h-10 w-full" />` |
| `/charts` | Skeleton chart area: `<Skeleton className="h-64 w-full" />` × 2 (one per chart section) |

### Error States

All three pages: server component fetch failures render an inline error banner:

```
<div role="alert" className="rounded-md border border-red-200 bg-red-50 p-4 text-sm text-red-700">
  {error message from Copywriting Contract}
</div>
```

Modal server errors: inline `<p role="alert" className="text-sm text-red-600">` below form fields, above DialogFooter.

## Accessibility Contracts

| Element | Requirement |
|---------|-------------|
| Nav links (new) | `min-h-[44px]` touch target — same as existing nav items |
| Icon-only action buttons | `aria-label` required on every icon button (e.g. `aria-label="Edit alert rule"`, `aria-label="Delete alert rule"`) |
| Pending spinners | `aria-label="Saving"` on `<Loader2>` — do not rely on animation alone |
| Modal dialogs | `<DialogTitle>` and `<DialogDescription>` required — radix Dialog handles aria-modal and focus trap |
| Alert/error regions | `role="alert"` on all inline error paragraphs and error banners |
| Chart container | `aria-label="Entry volume chart"` on `<ChartContainer>` wrapper |
| Table | No additional ARIA needed — shadcn Table uses semantic `<table>` elements |
| Badge | No interactive role — purely decorative; screen reader reads text content |
| Source filter dropdown | `aria-label="Filter by source"` on the `<Select>` trigger |
| Date range dropdown | `aria-label="Select date range"` on the `<Select>` trigger |
