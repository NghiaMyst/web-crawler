| Token | Value | Usage |

## Color

All values from the existing `globals.css` CSS variable system. Phase 9 adds **no new brand colors**. The three connection state colors are semantic status indicators using Tailwind core utilities.

| Role | Value (light mode) | Value (dark mode) | Usage |
|------|-------------------|-------------------|-------|
| Dominant (60%) | `--background: oklch(1 0 0)` (white) | `oklch(0.145 0 0)` (near-black) | Page background, main content area |
| Secondary (30%) | `--sidebar: oklch(0.985 0 0)`, `bg-zinc-100` | `oklch(0.205 0 0)` | Sidebar, mobile nav header, cards |
| Accent (10%) | `--primary: oklch(0.205 0 0)` (near-black) | `oklch(0.922 0 0)` (near-white) | Active nav links, primary buttons only |
| Destructive | `--destructive: oklch(0.577 0.245 27.325)` (red) | `oklch(0.704 0.191 22.216)` | Destructive action buttons only — none in Phase 9 |

**Connection State Color System** (semantic status — not in the 60/30/10 split):

| State | Tailwind Class | Approximate Hex | Meaning |
|-------|---------------|----------------|---------|
| Connected | `bg-green-500` | #22c55e | Hub connected and receiving events |
| Reconnecting | `bg-yellow-400` | #facc15 | Attempting to reconnect (pulsing) |
| Disconnected | `bg-red-500` | #ef4444 | Hub permanently disconnected |

These three colors are used exclusively on the `ConnectionDot` component. They are Tailwind core utilities (available in v4 without any config). They are not added to `globals.css` CSS variables because they are not brand colors — they are semantic status indicators.

Accent reserved for: primary action buttons (`Button variant="default"`) and active nav link highlight. The `ConnectionDot` states use their own semantic color system above, not the accent color.

Source: `apps/dashboard/app/globals.css` + `09-RESEARCH.md` Pattern 3.

## Interaction Contracts

### ConnectionDot — Visual State Machine

| State | Visual | Animation | `aria-label` |
|-------|--------|-----------|--------------|
| `HubConnectionState.Connected` | Green dot (`bg-green-500`) | None (static) | "Connected" |
| `HubConnectionState.Reconnecting` | Yellow dot (`bg-yellow-400`) | `animate-pulse` (Tailwind core) | "Reconnecting" |
| `HubConnectionState.Disconnected` | Red dot (`bg-red-500`) | None (static) | "Disconnected" |
| Initial render (before hub starts) | Red dot (`bg-red-500`) | None | "Disconnected" |

- Dot dimensions: `w-2 h-2` (8×8px), `rounded-full`
- Placement in Sidebar: `inline-flex items-center gap-1.5` container around existing `<span>Web Crawler</span>` + new `<ConnectionDot />`
- Placement in MobileNav: same inline-flex pattern next to the existing `<span className="font-semibold text-zinc-900">Web Crawler</span>`
- The dot itself is `inline-block` — no wrapper div needed; it sits inline with the text

### Sonner Toast — Trigger and Content

| Event | Toast variant | Copy | Position | Duration |
|-------|--------------|------|----------|----------|
| `onreconnected` (after gap recovery) | `toast.success()` | `"Reconnected — loaded N missed entries"` (N = exact count from gap recovery fetch) | bottom-right | 4000ms (Sonner default) |
| `onclose` / permanent disconnect | `toast.error()` | `"Live updates disconnected"` | bottom-right | persists until dismissed (Sonner error default) |

`<Toaster position="bottom-right" />` — bottom-right avoids overlapping table header filter controls (which are top-aligned).

Source: `09-CONTEXT.md` D-05 + `09-RESEARCH.md` Pattern 4 + Claude's discretion for position/duration.

### LiveEntriesWrapper — Data Flow

| Behavior | Contract |
|----------|----------|
| New entry received | Prepend to `liveEntries` array, re-render merged array |
| Row cap | `liveEntries.length + serverEntries.length > 200` → trim oldest live entries so total ≤ 200 |
| Reconnect gap recovery | Call `GET /api/entries?since=<last_received_at>` → prepend missed rows → show count in toast |
| No live entries yet | Render `<EntriesTable entries={serverEntries} />` unchanged (no empty state needed) |
| Hub not connected | Subscribe `useEffect` skips (`if (!connection) return`) — table shows server-fetched data only |

No new loading skeleton is needed — `LiveEntriesWrapper` renders the existing `EntriesTable` immediately with server data; live entries stream in after.

### Auto-Reconnect Policy

| Attempt | Delay |
|---------|-------|
| 1st | 0ms (immediate) |
| 2nd | 2000ms |
| 3rd | 10000ms |
| 4th | 30000ms |
| After 4th | `onclose` fires — permanent disconnect; toast shown |

Config: `.withAutomaticReconnect([0, 2000, 10000, 30000])` — satisfies SC-2 (auto-reconnect within 30s).

Source: `09-RESEARCH.md` Pattern 1 + Claude's discretion for retry intervals.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | No new CTA in Phase 9 — all interactions are automatic (connection manages itself) |
| Reconnect success toast | `"Reconnected — loaded N missed entries"` (N = integer from gap recovery count) |
| Reconnect zero-miss variant | `"Reconnected — no missed entries"` (when gap recovery returns 0 rows) |
| Disconnect toast | `"Live updates disconnected"` |
| ConnectionDot aria-label (connected) | `"Connected"` |
| ConnectionDot aria-label (reconnecting) | `"Reconnecting"` |
| ConnectionDot aria-label (disconnected) | `"Disconnected"` |
| Empty live entries | No empty state copy — server-fetched entries still render; silence is correct |
| Error: hub URL unreachable | No user-visible error beyond the disconnect toast — `onclose` fires, toast shows `"Live updates disconnected"` |

No destructive actions in Phase 9 — no confirmation dialogs needed.

Source: `09-CONTEXT.md` D-05 specifics + Claude's discretion for zero-miss variant.
