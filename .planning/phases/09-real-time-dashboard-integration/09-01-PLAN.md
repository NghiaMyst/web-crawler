```typescript
export const HubConnectionState = {
  Disconnected: 'Disconnected',
  Connected: 'Connected',
  Reconnecting: 'Reconnecting',
  Connecting: 'Connecting',
  Disconnecting: 'Disconnecting',
} as const;

export function createMockConnection() {
  const handlers: Record<string, Function[]> = {};
  let reconnectedCb: Function | null = null;
  let reconnectingCb: Function | null = null;
  let closeCb: Function | null = null;

  return {
    on: vi.fn((event: string, handler: Function) => {
      if (!handlers[event]) handlers[event] = [];
      handlers[event].push(handler);
    }),
    off: vi.fn((event: string, handler: Function) => {
      if (handlers[event]) {
        handlers[event] = handlers[event].filter((h) => h !== handler);
      }
    }),
    onreconnected: vi.fn((cb: Function) => { reconnectedCb = cb; }),
    onreconnecting: vi.fn((cb: Function) => { reconnectingCb = cb; }),
    onclose: vi.fn((cb: Function) => { closeCb = cb; }),
    start: vi.fn(() => Promise.resolve()),
    stop: vi.fn(() => Promise.resolve()),
    state: HubConnectionState.Disconnected,
    // Test helpers to simulate events:
    _emit(event: string, ...args: unknown[]) {
      handlers[event]?.forEach((h) => h(...args));
    },
    _simulateReconnected() { reconnectedCb?.(); },
    _simulateReconnecting() { reconnectingCb?.(); },
    _simulateClose() { closeCb?.(); },
  };
}
```

Per D-01, this is a global React Context Provider. Per RESEARCH Pitfall 1, store the
connection in `useState` (NOT `useRef`) so child components re-render when the connection

The file must contain:

2. Imports:
   - `createContext, useContext, useEffect, useState, useCallback, useRef` from `'react'`
   - `HubConnection, HubConnectionBuilder, HubConnectionState, LogLevel` from `'@microsoft/signalr'`
   - `toast` from `'sonner'`

3. `SignalRContextValue` interface:
   ```typescript
   interface SignalRContextValue {
     connection: HubConnection | null;
     state: HubConnectionState;
   }
   ```

4. Context with default value:
   ```typescript
   const SignalRContext = createContext<SignalRContextValue>({
     connection: null,
     state: HubConnectionState.Disconnected,
   });
   ```

5. `SignalRProvider` component:
   - Props: `{ children: React.ReactNode }`
   - State: `const [connection, setConnection] = useState<HubConnection | null>(null);`
   - State: `const [state, setState] = useState<HubConnectionState>(HubConnectionState.Disconnected);`
   - Ref for gap recovery callback: `const onReconnectRef = useRef<(() => Promise<void>) | null>(null);`
   - `useEffect` (mount/cleanup):
     - Build hub URL: `${process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:5000'}/hubs/dashboard`
     - Create connection: `new HubConnectionBuilder().withUrl(hubUrl).withAutomaticReconnect([0, 2000, 10000, 30000]).configureLogging(LogLevel.Warning).build()`
     - Register callbacks BEFORE `start()`:
       - `conn.onreconnecting(() => setState(HubConnectionState.Reconnecting))`
       - `conn.onreconnected(async () => { setState(HubConnectionState.Connected); const gapFn = onReconnectRef.current; if (gapFn) { try { await gapFn(); } catch { /* gap recovery best-effort */ } } })`
       - `conn.onclose(() => { setState(HubConnectionState.Disconnected); toast.error('Live updates disconnected'); })`
     - Set connection in state: `setConnection(conn);`
     - Start: `conn.start().then(() => setState(HubConnectionState.Connected)).catch(() => setState(HubConnectionState.Disconnected))`
     - Cleanup: `return () => { conn.stop(); }`
   - Dependency array: `[]` (run once on mount)
   - Render: `<SignalRContext.Provider value={{ connection, state }}>{children}</SignalRContext.Provider>`

6. `useSignalRContext` hook:
   ```typescript
   export function useSignalRContext(): SignalRContextValue {
     return useContext(SignalRContext);
   }
   ```

   (specifically `LiveEntriesWrapper` in Plan 09-02) to register a reconnect callback

   ```typescript
   interface SignalRContextValue {
     connection: HubConnection | null;
     state: HubConnectionState;
     registerReconnectHandler: (fn: (() => Promise<void>) | null) => void;
   }
   ```
   And implement `registerReconnectHandler` as:
   ```typescript
   const registerReconnectHandler = useCallback((fn: (() => Promise<void>) | null) => {
     onReconnectRef.current = fn;
   }, []);
   ```
   Update the default context value to include `registerReconnectHandler: () => {}`.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SignalR hub -> browser client | Server pushes `NewEntry` payloads over WebSocket; payload originates from .NET API (trusted) |
| Browser -> NEXT_PUBLIC_API_URL | Client constructs hub URL from environment variable exposed to browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | NewEntry payload from hub | accept | Payload originates from trusted .NET API server; no user input involved; `DataEntry` type assertion provides shape safety; single-user personal project |
| T-09-02 | Information Disclosure | NEXT_PUBLIC_API_URL in client bundle | accept | API URL is intentionally public (browser must reach it); no secrets in URL; standard Next.js pattern for public env vars |
| T-09-03 | Denial of Service | Unbounded liveEntries array growth | mitigate | ROW_CAP = 200 enforced in LiveEntriesWrapper (Plan 09-02); prevents memory exhaustion from rapid NewEntry events |
| T-09-04 | Spoofing | CORS bypass on hub connection | accept | Server-side concern handled in Phase 6 D-07 (`AllowCredentials()` with explicit origins); no client-side mitigation needed |
</threat_model>
