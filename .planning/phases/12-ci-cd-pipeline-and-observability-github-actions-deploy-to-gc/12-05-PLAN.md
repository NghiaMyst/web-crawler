    **Sub-task B: Update nginx/nginx.conf**

    Open `nginx/nginx.conf`. Make two additions, both in the `http {}` block:

    **Addition 1 — New map block** for Grafana Live WebSocket. Add AFTER the existing SignalR map block (after line 32):

    The existing block is:
    ```nginx
        map $http_connection $connection_upgrade {
            "~*Upgrade" $http_connection;
            default     keep-alive;
        }
    ```

    Add immediately after:
    ```nginx
        # Grafana Live WebSocket upgrade — uses DIFFERENT output variable to avoid nginx
        # "duplicate map variable" error with the SignalR map above.
        map $http_upgrade $grafana_connection_upgrade {
            default upgrade;
            ''      close;
        }
    ```

    **Addition 2 — Location blocks for Grafana** inside the `server { listen 443 ssl; ... }` block. Add BEFORE the existing `location /hubs/` block:

    ```nginx
            # Grafana UI — proxied from /grafana/ subpath.
            # Basic auth provided by nginx; Grafana's own login is secondary.
            location /grafana/ {
                auth_basic "Grafana";
                auth_basic_user_file /etc/nginx/.htpasswd;
                proxy_set_header Host $host;
                proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
                proxy_set_header X-Forwarded-Proto $scheme;
                proxy_pass http://grafana:3000;
                rewrite ^/grafana/(.*) /$1 break;
            }

            # Grafana Live — WebSocket for real-time dashboard updates.
            location /grafana/api/live/ {
                auth_basic "Grafana";
                auth_basic_user_file /etc/nginx/.htpasswd;
                proxy_http_version 1.1;
                proxy_set_header Upgrade $http_upgrade;
                proxy_set_header Connection $grafana_connection_upgrade;
                proxy_set_header Host $host;
                proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
                proxy_set_header X-Forwarded-Proto $scheme;
                proxy_pass http://grafana:3000;
                rewrite ^/grafana/(.*) /$1 break;
            }
    ```

    **Do NOT modify** the existing SignalR `map $http_connection $connection_upgrade` block.
    **Do NOT modify** the existing `/hubs/` or `/` location blocks.
    **Do NOT modify** the `server { listen 80; }` redirect block.

    The key requirement: `$grafana_connection_upgrade` (not `$connection_upgrade`) is used in the Grafana Live location block. This avoids the `duplicate map variable` nginx startup error.
