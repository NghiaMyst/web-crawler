<task type="auto">
  <name>Task 2: Add deploy job to .github/workflows/deploy.yml</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
    Open `.github/workflows/deploy.yml` (created in Plan 02). The file currently has only the `build-and-push` job. Add the `deploy` job after it by replacing the trailing comment `# deploy job is added in Plan 03 (depends on build-and-push completing)` with the full deploy job definition:

    ```yaml
      deploy:
        runs-on: ubuntu-latest
        needs: build-and-push
        steps:
          - name: SSH deploy to GCE VM
            uses: appleboy/ssh-action@v1.2.5
            env:
              IMAGE_TAG: ${{ github.sha }}
            with:
              host: ${{ secrets.GCE_SSH_HOST }}
              username: ${{ secrets.GCE_SSH_USER }}
              key: ${{ secrets.GCE_SSH_PRIVATE_KEY }}
              fingerprint: ${{ secrets.GCE_SSH_KNOWN_HOSTS }}
              envs: IMAGE_TAG
              command_timeout: 10m
              script: |
                set -e
                cd /opt/webcrawler

                # Update IMAGE_TAG in .env.prod in-place (preserves all other secrets)
                sed -i "s|^IMAGE_TAG=.*|IMAGE_TAG=${IMAGE_TAG}|" .env.prod
                grep -q "^IMAGE_TAG=" .env.prod || echo "IMAGE_TAG=${IMAGE_TAG}" >> .env.prod

                # Pull new images for crawler and api only (skip postgres/redis/nginx)
                docker compose -f docker-compose.prod.yml --env-file .env.prod pull crawler api

                # Restart only crawler and api (--no-deps skips postgres/redis/nginx restart)
                docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --no-deps crawler api

                # Wait for API healthcheck (up to 90 seconds)
                echo "Waiting for API healthcheck..."
                timeout 90 bash -c 'until curl -sf http://localhost:5000/health > /dev/null; do echo "  still waiting..."; sleep 5; done'

                echo "Deploy SUCCESS: IMAGE_TAG=${IMAGE_TAG}"
    ```

    **Critical details in the deploy script:**
    - `set -e` — fail fast on any error (prevents silent deploy failures)
    - `sed -i "s|^IMAGE_TAG=.*|IMAGE_TAG=${IMAGE_TAG}|"` — updates existing IMAGE_TAG line in-place. Uses `|` as delimiter to avoid conflicts with path characters in the value
    - `grep -q ... || echo ...` — adds IMAGE_TAG on first deploy when the key does not exist yet in `.env.prod`
    - `pull crawler api` — pulls only crawler and api images (Prometheus/Grafana/postgres/redis/nginx use stock images from Docker Hub, not Artifact Registry)
    - `up -d --no-deps crawler api` — restarts only crawler and api without restarting postgres/redis/nginx
    - `timeout 90` — gives the API 90 seconds to become healthy (accounts for DB migration on startup)
    - `command_timeout: 10m` — gives the entire deploy step 10 minutes (pull may take 2-3 min for large images)
    - `needs: build-and-push` — deploy only runs after ALL matrix jobs in build-and-push succeed

    The final workflow file should have exactly two jobs: `build-and-push` and `deploy`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GCE VM | SSH connection using Ed25519 deploy key stored as GitHub Secret |
| SSH deploy script → .env.prod | sed modifies a single key in-place; does not read or echo other secret values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-01 | Tampering | .env.prod modification via sed | mitigate | `sed -i "s|^IMAGE_TAG=.*|IMAGE_TAG=${IMAGE_TAG}|"` uses anchored regex that only matches lines starting with `IMAGE_TAG=`. Other secrets are structurally unreachable by this pattern. `set -e` ensures the script fails if sed errors. |
| T-12-03-02 | Denial of Service | deploy script pulls wrong image | accept | Images are tagged with `github.sha` (immutable, unique per commit). Rolling back is `sed -i "s|^IMAGE_TAG=.*|IMAGE_TAG=<old-sha>|" .env.prod && docker compose up -d --no-deps crawler api`. |
| T-12-03-03 | Information Disclosure | IMAGE_TAG exposed via docker inspect | accept | `github.sha` is public information (visible in the GitHub commit log). No secrets are exposed. |
| T-12-03-04 | Spoofing | SSH host key forgery | mitigate | `fingerprint: ${{ secrets.GCE_SSH_KNOWN_HOSTS }}` (the known_hosts content) passed to appleboy/ssh-action prevents MITM attacks. Known_hosts stored as GitHub Secret, verified once via `ssh-keyscan` during setup. |
</threat_model>
