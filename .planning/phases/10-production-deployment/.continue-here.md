# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through code review.
> Acknowledge each one explicitly before proceeding to production deploy.

- [ ] CONSTRAINT: CR-01 — `apps/api/.env.prod.example` uses `ConnectionStrings__DefaultConnection` but `apps/api/Program.cs` reads `DATABASE_URL`. Any operator following the runbook will get an API container crash at startup. Fix the key name before deploying.
- [ ] CONSTRAINT: CR-02 — `apps/crawler/src/index.ts` calls `await saveBloomFilter()` in `additionalCleanup` with no try/catch. If Redis is unavailable at shutdown, the rejection aborts cleanup and skips `browserPool.closeAll()` and all six worker `.close()` calls — Playwright subprocesses leak. Wrap the call in try/catch.

**Do not proceed to production deploy until both are fixed.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Mismatched env key | `.env.prod.example` documents `ConnectionStrings__DefaultConnection` but `Program.cs` hard-reads `DATABASE_URL` — operator crash on first deploy | blocking | Read `Program.cs` GetEnvironmentVariable calls before writing example env file |
| Unguarded async in shutdown | `saveBloomFilter()` called without try/catch in shutdown callback — Redis error aborts remaining cleanup | blocking | Any async call in a shutdown/cleanup path must be wrapped in try/catch |
