# Plan 02: Per-Domain Politeness Guard

## Goal
Create the Redis-based politeness guard that enforces a minimum 2-second delay between requests to the same domain (per D-01).

## Context
This is a Wave 1 plan running parallel to Plans 01, 03, and 04. It creates a standalone service module that will be wired into crawlWorker.ts in Plan 05. Per D-01, this uses Redis timestamps (NOT per-domain BullMQ queues).

<instructions>
1. Create `apps/crawler/src/services/politenessGuard.ts` per D-01 and RESEARCH.md Pattern 2:
   - Import `connection` from `../connection.js`
   - Export constant `POLITENESS_DELAY_MS = 2000`
   - Export constant `KEY_TTL_S = 10` (short TTL -- only needed for ~2s check window)
   - Export async function `enforcePoliteness(domain: string): Promise<void>`:
     - Build key: `crawl:politeness:${domain}`
     - GET key from Redis
     - If exists, compute elapsed = `Date.now() - parseInt(lastTs, 10)`
     - If elapsed < POLITENESS_DELAY_MS, await `setTimeout` for the remainder
     - After any delay, SET key to `Date.now().toString()` with EX KEY_TTL_S
     - This records the ACTUAL dispatch time, not the check time

2. Create `apps/crawler/src/services/politenessGuard.test.ts`:
   - Mock `../connection.js` using `vi.mock` -- mock `connection.get` and `connection.set`
   - Test 1: When Redis returns null (no previous timestamp), function resolves without waiting and calls `connection.set`
   - Test 2: When Redis returns a timestamp from 500ms ago, function waits ~1500ms before resolving (use `vi.useFakeTimers` or measure actual elapsed time)
   - Test 3: When Redis returns a timestamp from 3000ms ago (>2s elapsed), function resolves immediately
   - Test 4: Verify `connection.set` is called with key matching `crawl:politeness:` prefix and 'EX' and 10

3. Run tests: `pnpm --filter @web-crawler/crawler test`
</instructions>
