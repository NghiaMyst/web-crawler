# Plan 01: Bloom Filter URL Deduplication

## Goal
Install Bloom Filter dependency and Vitest, create the in-memory Bloom Filter singleton service with unit tests.

## Context
This is the first plan in Phase 2. It establishes the test infrastructure (Vitest) used by all subsequent plans, and creates the Bloom Filter service that will be wired into crawlWorker.ts in Plan 05. Per D-02, the filter is in-memory only -- no Redis persistence (deferred to Phase 10).

<instructions>
1. Install dependencies:
   ```bash
   pnpm --filter @web-crawler/crawler add bloom-filters
   pnpm --filter @web-crawler/crawler add -D vitest
   ```

2. Create `apps/crawler/vitest.config.ts`:
   ```typescript
   import { defineConfig } from 'vitest/config';
   export default defineConfig({
     test: {
       globals: false,
       environment: 'node',
     },
   });
   ```

3. Add test script to `apps/crawler/package.json`:
   ```json
   "test": "vitest run --reporter=verbose"
   ```

4. Create `apps/crawler/src/services/bloomFilter.ts` -- per D-02:
   - Import `BloomFilter` from `bloom-filters` (ESM import works per RESEARCH.md verification)
   - Create singleton: `export const bloomFilter = BloomFilter.create(100000, 0.01)` (100k capacity, 1% FP per D-02)
   - Export `isUrlSeen(url: string): boolean` -- calls `bloomFilter.has(url)`
   - Export `markUrlSeen(url: string): void` -- calls `bloomFilter.add(url)`
   - NO Redis persistence -- D-02 explicitly defers to Phase 10

5. Create `apps/crawler/src/services/bloomFilter.test.ts`:
   - Import `{ describe, it, expect }` from `vitest`
   - Import `{ bloomFilter, isUrlSeen, markUrlSeen }` from `./bloomFilter.js`
   - Test: unseen URL returns false
   - Test: after markUrlSeen, same URL returns true
   - Test: different URL still returns false
   - Test: bloomFilter is defined (singleton exists)

6. Run tests: `pnpm --filter @web-crawler/crawler test`
</instructions>
