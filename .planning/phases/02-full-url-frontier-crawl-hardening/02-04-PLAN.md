# Plan 04: MD5 Content Hash Deduplication

## Goal
Create the content hash deduplication service that compares MD5 hashes of response bodies per sourceId, stored in Redis (per D-04).

## Context
Wave 1 plan running parallel to Plans 01, 02, 03. Uses only Node.js built-in `crypto` -- no extra dependency. Will be wired into crawlWorker.ts in Plan 05 as a post-fetch guard.

<instructions>
1. Create `apps/crawler/src/services/contentHash.ts` per D-04 and RESEARCH.md Pattern 4:
   - Import `{ createHash }` from `node:crypto`
   - Import `{ connection }` from `../connection.js`
   - Export async function `isContentChanged(sourceId: string, body: string): Promise<boolean>`:
     - Compute hash: `createHash('md5').update(body).digest('hex')`
     - Build Redis key: `crawl:hash:${sourceId}`
     - GET previous hash from Redis
     - If `prevHash === hash`, return false (unchanged)
     - Otherwise, SET new hash in Redis (no TTL -- persists until overwritten)
     - Return true (changed)

2. Create `apps/crawler/src/services/contentHash.test.ts`:
   - Mock `../connection.js` -- mock `connection.get` and `connection.set`
   - Test 1: No previous hash (Redis returns null) -- returns true, calls `connection.set` with the MD5 hex
   - Test 2: Previous hash matches current body hash -- returns false, does NOT call `connection.set`
   - Test 3: Previous hash differs from current body hash -- returns true, calls `connection.set` with new hash
   - Test 4: Same body but different sourceId -- returns true (different Redis key)
   - Verify the MD5 hex of a known string matches expected value (e.g., MD5 of 'hello' = '5d41402abc4b2a76b9719d911017c592')

3. Run tests: `pnpm --filter @web-crawler/crawler test`
</instructions>

### Must-Haves

- `isContentChanged` returns false when body hash matches stored hash
- `isContentChanged` returns true and updates Redis when body hash differs
