# Plan 03: robots.txt Fetcher and Cache

## Goal
Install `robots-parser`, create the robots.txt fetcher/cache service that checks URL allowance per D-03, with Redis caching and 24h TTL.

## Context
Wave 1 plan running parallel to Plans 01, 02, and 04. Creates a standalone service wired into crawlWorker.ts in Plan 05. Per D-03, disallowed URLs are logged with `disallowed` status and skipped entirely.

<instructions>
1. Install dependency:
   ```bash
   pnpm --filter @web-crawler/crawler add robots-parser
   ```

2. Create `apps/crawler/src/services/robotsCache.ts` per D-03 and RESEARCH.md Pattern 3:
   - Import `axios` from `axios`
   - Import `robotsParser` from `robots-parser` (works with `esModuleInterop: true` per RESEARCH.md Pitfall 2)
   - Import `{ connection }` from `../connection.js`
   - Constants: `CACHE_TTL_S = 86400` (24h), `USER_AGENT = 'PersonalCrawlerBot/1.0'`
   - Export async function `isUrlAllowed(url: string): Promise<boolean>`:
     - Parse hostname and protocol from `new URL(url)`
     - Build Redis key: `crawl:robots:${hostname}`
     - GET from Redis
     - If cache miss: fetch `${protocol}//${hostname}/robots.txt` with axios (timeout 5000, responseType 'text', User-Agent header)
     - On fetch error (404, timeout, network): set `robotsTxt = ''` (permissive default)
     - SET in Redis with EX CACHE_TTL_S
     - Parse with `robotsParser(`${protocol}//${hostname}/robots.txt`, robotsTxt)`
     - Return `robots.isAllowed(url, USER_AGENT) !== false` (treat undefined as allowed)

3. Create `apps/crawler/src/services/robotsCache.test.ts`:
   - Mock `../connection.js` -- mock `connection.get` and `connection.set`
   - Mock `axios` -- mock `axios.get`
   - Test 1: Disallowed URL -- mock Redis returns null (cache miss), mock axios returns `User-agent: *\nDisallow: /secret`, call `isUrlAllowed('https://example.com/secret')` -- expect false
   - Test 2: Allowed URL -- same robots.txt, call `isUrlAllowed('https://example.com/public')` -- expect true
   - Test 3: Cached robots.txt -- mock Redis returns the robots.txt string, verify axios.get NOT called
   - Test 4: Fetch failure -- mock axios.get to throw, verify function returns true (permissive default)
   - Test 5: Verify `connection.set` called with key `crawl:robots:example.com`, the robots.txt content, 'EX', 86400

4. Run tests: `pnpm --filter @web-crawler/crawler test`
</instructions>

## Threat Model

| Boundary | Description |
|----------|-------------|
| SSRF via robots.txt fetch | The `${protocol}//${hostname}/robots.txt` URL is derived from crawl target URLs |

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-02-03 | Spoofing / SSRF | robotsCache.ts | mitigate | Only fetch `robots.txt` at the exact hostname derived from the crawl URL via `new URL(url).hostname`; timeout at 5s; no user-controlled path component in the fetch URL |
| T-02-04 | Information Disclosure | robotsCache.ts | accept | robots.txt content cached in Redis; personal project, Redis not exposed externally |

### Must-Haves
- `isUrlAllowed` returns false for disallowed paths
- `isUrlAllowed` returns true for allowed/unmentioned paths
- Redis cache with 24h TTL prevents repeated robots.txt fetches
- Failed robots.txt fetch defaults to permissive (allow)
