# Plan 06: Remaining Data Source Workers

## Goal
Create four new source workers (Genshin, LoL, AniList, MangaDex) following the established FootballDataWorker pattern, each with their own BullMQ queue and scheduler, all wired into index.ts.

## Context
Wave 3 plan -- depends on Plan 05 (guards are wired). Per D-07, each source gets a dedicated worker class and BullMQ queue. Phase 2 goal is raw crawl output in logs only -- no parsing, no storage. All four workers follow the `FootballDataWorker.ts` pattern exactly: axios client, BullMQ worker, error logging, re-throw on failure.

**Queue files** -- each follows `footballDataQueue.ts` pattern exactly. Queue names use kebab-case, NO colons (BullMQ v5 restriction):

1. `apps/crawler/src/queues/genshinQueue.ts` -- queue name: `crawl-genshin`
2. `apps/crawler/src/queues/lolQueue.ts` -- queue name: `crawl-lol`

4. `apps/crawler/src/queues/mangadexQueue.ts` -- queue name: `crawl-mangadex`

Each queue file:
```typescript
import { Queue } from 'bullmq';
import { connection } from '../connection.js';

export const {name}Queue = new Queue('crawl-{name}', {
  connection,
  defaultJobOptions: {
    attempts: 3,
    backoff: { type: 'exponential', delay: 5000 },
    removeOnComplete: { count: 100 },
    removeOnFail: { count: 500 },
  },
});
```

**Worker files** -- each follows `FootballDataWorker.ts` pattern:

5. `apps/crawler/src/workers/GenshinWorker.ts` (SRC-02):
   - SOURCE_ID: `'hoyowiki-genshin'`
   - Job handler tries HoYoWiki API first: `GET https://sg-wiki-api-static.hoyolab.com/hoyowiki/genshin/wapi/home` with User-Agent header
   - If API fails, fallback to `cheerioFetch('https://www.hoyolab.com/circles/2/41/official', ...)` from CheerioWorker.ts
   - Log raw JSON or HTML response with `logger.info('Genshin raw response', { sourceId, jobId, data })`
   - Export `createGenshinWorker()` returning `Worker<GenshinJobData>`

6. `apps/crawler/src/workers/LoLWorker.ts` (SRC-03):
   - SOURCE_ID: `'lol-tierlist'`
   - Job handler fetches `https://u.gg/lol/tier-list` via axios (NOT cheerioFetch -- we need raw HTML for __NEXT_DATA__ extraction)
   - Extract `__NEXT_DATA__`:
     ```typescript
     import * as cheerio from 'cheerio';
     const $ = cheerio.load(html);
     const scriptContent = $('script#__NEXT_DATA__').text();
     const nextData = JSON.parse(scriptContent) as Record<string, unknown>;
     ```
   - Log raw data: `logger.info('u.gg tier list raw data', { sourceId, jobId, dataKeys: Object.keys(nextData.props ?? {}) })`
   - Also check `RIOT_API_KEY` env var at import time. If missing, log warning but do NOT crash. The u.gg scrape works without it.
   - Export `createLoLWorker()` returning `Worker<LoLJobData>`

7. `apps/crawler/src/workers/AniListWorker.ts` (SRC-04):
   - SOURCE_ID: `'anilist'`
   - Job handler POSTs GraphQL query to `https://graphql.anilist.co`:
     ```graphql
     query {
       Page(page: 1, perPage: 50) {
         media(type: ANIME, status: RELEASING, sort: POPULARITY_DESC) {
           id
           title { romaji english }
           nextAiringEpisode { airingAt episode }
         }
       }
     }
     ```
   - Use axios.post with headers: `Content-Type: application/json`, `User-Agent: PersonalCrawlerBot/1.0`
   - Timeout: 10_000ms
   - Log raw response: `logger.info('AniList raw response', { sourceId, jobId, mediaCount: data.data.Page.media.length })`
   - NOTE: Do NOT hardcode seasonYear -- use RELEASING status + POPULARITY_DESC sort (works across seasons)
   - Export `createAniListWorker()` returning `Worker<AniListJobData>`

8. `apps/crawler/src/workers/MangaDexWorker.ts` (SRC-05):
   - SOURCE_ID: `'mangadex'`
   - Job handler GETs `https://api.mangadex.org/chapter?order[publishAt]=desc&limit=10` with User-Agent header
   - Log raw response: `logger.info('MangaDex raw response', { sourceId, jobId, chapterCount: data.data.length })`
   - Timeout: 10_000ms
   - Export `createMangaDexWorker()` returning `Worker<MangaDexJobData>`

**All workers must:**
- Import `connection` from `../connection.js`
- Import `logger` from `../logger.js`
- Set `concurrency: 1` in worker options
- Have `completed` and `failed` event handlers matching FootballDataWorker pattern
- catch -> log with logger.error -> re-throw (BullMQ retry pattern)
- Set `User-Agent: PersonalCrawlerBot/1.0` on all HTTP requests

   After the football scheduler, register schedulers for each source using `upsertJobScheduler` (same pattern as football):
   - Genshin: every 6 hours (`6 * 60 * 60 * 1000`), scheduler ID: `genshin-events-scheduler`, job name: `fetch-genshin-events`
   - LoL: every 12 hours (`12 * 60 * 60 * 1000`), scheduler ID: `lol-tierlist-scheduler`, job name: `fetch-lol-tierlist`
   - AniList: every 6 hours (`6 * 60 * 60 * 1000`), scheduler ID: `anilist-airing-scheduler`, job name: `fetch-anilist-airing`
   - MangaDex: every 1 hour (`60 * 60 * 1000`), scheduler ID: `mangadex-chapters-scheduler`, job name: `fetch-mangadex-chapters`

   Each scheduler call follows the football pattern:
   ```typescript
   await {queue}.upsertJobScheduler(
     '{scheduler-id}',
     { every: {interval_ms} },
     {
       name: '{job-name}',
       data: {},
       opts: {
         attempts: 3,
         backoff: { type: 'exponential', delay: 5000 },
       },
     },
   );
   ```

   Update graceful shutdown to close all new workers:
   ```typescript
   await setupGracefulShutdown(crawlWorker, async () => {

     await footballWorker.close();
     await genshinWorker.close();
     await lolWorker.close();
     await anilistWorker.close();
     await mangadexWorker.close();
   });
   ```

## Threat Model

| Boundary | Description |
|----------|-------------|
| External API calls | Workers make HTTP requests to third-party APIs |
| API key exposure | RIOT_API_KEY must not be logged or committed |

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-02-08 | Information Disclosure | LoLWorker.ts | mitigate | RIOT_API_KEY read from env var, never logged; Docker Compose uses variable substitution not hardcoded value |
| T-02-09 | Denial of Service (to target) | All workers | mitigate | Each worker runs concurrency:1; source-specific workers do not bypass politeness guard for their own domains; scheduler intervals are conservative (1-12 hours) |
| T-02-10 | Spoofing (SSRF) | GenshinWorker.ts | accept | URLs are hardcoded in worker code, not user-controlled; only fetches known API endpoints |

### Must-Haves
- All four workers follow FootballDataWorker pattern exactly
- All four queues use kebab-case names with no colons
- All four workers registered in index.ts with schedulers
- Graceful shutdown closes all workers
- RIOT_API_KEY in Docker Compose env (defaults to empty)
- Type check passes with all new files
